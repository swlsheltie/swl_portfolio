PAGE IN PROGRESS

rethinking browsers—collaborative research

Bento Browser started as a way reimagine how individuals use web browsers, on desktop and mobile. How can we optimize search while decreasing cognitive load between search sessions on mobile browsers?

As for desktop, we started to ask whether or not the desktop browsing experience should have different features than mobile. Are tabbed browsers are really the best way to help us sort through all this information. Is there a better way to organize our web browsing?

As Bento Mobile took off, we wanted to see how it would translate to a desktop space. With so much more screen real estate, infinite possibilities open up.

This project is in progress! Prototypes and images shown here are all part of an ongoing research lab.

overview

problem space

Bento desktop is the reimagined desktop browser that helps you prioiritize work and research. Bento mobile is a mobile browser that surfaces and organizaes your recent searches so that you don’t need to remember your history while conducting quick reserach. As we conducted research, we settled upon several key points of focus. We centered our work around answering these specific questions.

desktop experience

➀ How can we re-imagine tabs and how far can we push a new mental model of organization?
➁ How can we facilitate research by smoothly integrating note-taking into the browser?
➂ If note-taking is a primary way of saving research, how can we further facilitate information organization by introducing a way to keep different projects + types of information separate?
➃ Can browsers be collaborative?

mobile experience

➀ How can we re-frame the browsing experience around research and annotations?
➁ Is collaboration a need? If so, how does it fit into the picture?

the desktop experience

➀ page queueing + queue manager

How can we re-imagine tabs and how far can we push a new mental model of organization?

Currently, our team has prototyped out what a possible new tab page could look like. This prototype shows possible interactions between mobile-desktop syncing, organization in the New Tab page, and the extension pop out. When opening a new tab, the "New Tab" page presents an opportunity for a moment of pause in the workflow, which we found to be an opportunity for users to take time to organize their tabs/notes/etc. Currently open tabs can be dragged into folders, and users are also able to switch between folder and list views.

Upon exiting the new tab page, users can also add whatever current webpage they are on to a folder from the extension pop out.
